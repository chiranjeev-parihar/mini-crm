# Follow-up Management Implementation Plan

This document outlines the proposed changes to implement the Follow-up Management module for the Mini CRM.

## User Review Required

> [!IMPORTANT]
> The database schema will be updated to include the `FollowUp` model. This requires running `npx prisma db push` or `npx prisma migrate dev`. I will execute the schema changes and migration after you approve this plan.

## Proposed Changes

---

### Database Layer

#### [MODIFY] [schema.prisma](file:///f:/Project_Ishita/Portfolio/MiniCRM/database/prisma/schema.prisma)
Add the `FollowUp` model, `FollowUpType` enum, and `FollowUpStatus` enum.
- Fields: `id`, `leadId`, `followUpDate`, `followUpTime`, `type`, `status`, `notes`, `outcome`, `nextFollowUpDate`, `createdBy`, `createdAt`, `updatedAt`
- Relations: Link `FollowUp` to `Lead` (many-to-one) and `User` (many-to-one for createdBy).

## Verification Plan

### Automated Tests
- Run `npm run build` in both frontend and backend to verify TypeScript compilations.

### Manual Verification
- Start the server and frontend.
- Go to Dashboard, verify widgets.
- Create a Lead.
- From Lead Details, schedule a Follow-up.
- Check Follow-up History in Lead Details.
- Go to Follow-ups list, edit the pending follow-up.
- Complete the follow-up, provide an outcome, and schedule the next follow-up.
- Attempt to edit the completed follow-up (should be prevented).
- Verify Missed follow-ups appear in red.
