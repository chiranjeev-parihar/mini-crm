# AI Development Rules

Always follow these rules:

1. Read all documentation inside the docs folder before generating code.
2. Follow the project architecture.
3. Never delete existing working code unless explicitly instructed.
4. Generate production-quality code.
5. Use TypeScript.
6. Follow SOLID principles.
7. Keep components small and reusable.
8. Never hardcode configuration values.
9. Validate all forms.
10. Handle loading, error, and empty states.
11. Explain which files are created or modified before generating code.
12. If requirements are unclear, ask instead of guessing.
13. Complete only the requested feature. Do not add unrelated features.
14. Ensure the project builds successfully after changes.
15. Follow the folder structure defined in the project documentation.
