# Authentication Module

## Purpose

Provide secure access to the Mini CRM using email and password authentication.

---

## User Roles

### Admin
- Full system access

### Sales Executive
- Manage assigned leads
- View dashboard
- Manage follow-ups

### Sales Manager
- Manage team
- Assign leads
- View reports

---

## User Stories

- As a user, I want to log in securely.
- As a user, I should only access protected pages after authentication.
- As a user, I should remain logged in until my token expires.
- As a user, I should be able to logout safely.

---

## Features

- Login
- Logout
- JWT Authentication
- Password Hashing
- Protected Routes
- Role Based Access

---

## API

POST /api/auth/login

POST /api/auth/logout

GET /api/auth/me

---

## Database

User

- id
- name
- email
- password
- role
- createdAt
- updatedAt

---

## Acceptance Criteria

- Login works
- Logout works
- Invalid credentials show errors
- JWT validates protected routes
- Passwords are hashed
